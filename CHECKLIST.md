## 🎓 Course Requirements

### Human Computer Interaction (HCI)

- [ ] Target users clearly identified.
- [ ] UX research conducted (interviews, surveys, observations, etc.).
- [ ] User pain points analyzed.
- [ ] At least one persona created.
- [ ] User stories defined for main features.
- [ ] User flow or journey documented.
- [ ] Wireframes or mockups created before implementation.
- [ ] Interface designed using usability and consistency principles.
- [ ] Clear navigation and user feedback implemented.

### Software Engineering

- [ ] Functional and non-functional requirements identified (Main Features).
- [ ] Development Methodology identified (SCRUM, Waterfall, etc.).
- [ ] UML Diagrams created:
  - [ ] Use Case Diagram (with descriptions).
  - [ ] Activity Diagram (with descriptions).
  - [ ] Class Diagram (with descriptions).
  - [ ] Sequence Diagram (with descriptions).
- [ ] Team responsibilities organized per user stories.
- [ ] Development planning demonstrated.

### Backend Development

- [ ] REST API implemented using Node.js and Express.js.
- [ ] API routes separated properly (routes/controllers/services/repositories).
- [ ] Authentication implemented using JWT tokens.
- [ ] Protected routes requiring authentication.
- [ ] Authorization / privilege management implemented (roles/permissions).
- [ ] API connected to a database.
- [ ] CRUD operations implemented for main resources.
- [ ] API tested using Postman, Swagger UI, or ApiDog.
- [ ] Swagger API documentation available with testable requests.
- [ ] Proper HTTP status codes used.
- [ ] Environment variables stored in `.env`.
- [ ] Error handling middleware implemented.
- [ ] Frontend integration with ReactJS.

### Database Administration

- [ ] Database implemented using MySQL (or Oracle, PostgreSQL, etc.).
- [ ] Database Design and Analysis:
  - [ ] ERD designed appropriately.
  - [ ] Relational Model (RM) designed appropriately.
  - [ ] Primary keys and foreign keys correctly implemented.
  - [ ] Relationships between entities properly designed.
  - [ ] Constraints implemented where relevant.
- [ ] SQL scripts provided for: table creation, insertion, and queries.
- [ ] Sample data available.
- [ ] Database connected successfully to the application.
- [ ] Meaningful queries demonstrated.
- [ ] User access and control management (User Privilege and Roles).
- [ ] Backup and recovery mechanisms (export scripts).
- [ ] Query optimization techniques (indexing implemented properly).
- [ ] Data encryption implemented where necessary.
